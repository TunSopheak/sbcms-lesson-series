Section 01

Wireframes — គ្រោង UI

Wireframe ជាការគ្រោង Layout នៃ App/Website ដោយមិនមានពណ៌, មិនមាន Image — ប្រើ Box, Line, Text ប្រហែលៗ ។ គោលបំណង: ផ្តោតលើ Structure & Function ជាមុន មុននឹង Design ពណ៌ ។

📐
ហេតុអ្វីត្រូវ Wireframe មុន?
ធ្វើ Wireframe មុន → ចំណាយពេលតិច → ប្រសិនបើ Layout ខុស កែបានលឿន ។ ប្រសិនបើ Code ហើយទើបដឹង Layout ខុស → ខ្ជះខ្ជាយពេល ។ Wireframe ជួយ Team និង Stakeholder ឯកភាពលើ Structure ជាមុន ។

✏️

🖥️

CSS Grid vs Flexbox

⊞
CSS Grid
ល្អសម្រាប់ 2D Layout — Rows & Columns ។ ប្រើធ្វើ Page Layout, Dashboard, Card Grid ។
display: grid
⇌
Flexbox
ល្អសម្រាប់ 1D Layout — Row ឬ Column ។ ប្រើធ្វើ Nav Bar, Button Row, Align Items ។
display: flex
CSS — Grid System Example (SBCMS Dashboard) 
/* SBCMS — 3-column Stats Card Grid */
.stats-row {
  display:               grid;
  grid-template-columns: repeat(3, 1fr);
  gap:                   14px;
  margin-bottom:        20px;
}

/* SBCMS — Sidebar + Main Content (8+4) */
.app-layout {
  display:  grid;
  grid-template-columns: 200px 1fr;  /* Fixed sidebar */
  height:   100vh;
}

/* POS Split Layout — Flexbox */
.pos-layout {
  display:   flex;
  gap:       0;
  height:    100%;
}
.menu-grid  { flex: 1; }         /* Takes remaining space */
.order-panel{ width: 280px; }    /* Fixed order panel */

Responsive Design — ការ Adapt ទៅ Device

Responsive Design ជា Technique ធ្វើឱ្យ UI Adapt ទៅតាម Screen Size — Desktop, Tablet, Mobile ។ User ទំនើប ប្រើ Device ច្រើនប្រភេទ ។

📱
SBCMS — Web vs Desktop
SBCMS Web App (ASP.NET MVC) ត្រូវ Responsive ព្រោះ Customer អាច Order ពី Mobile, Tablet ។ ផ្ទុយ Desktop App (C# WinForms) ជា Fixed-size Window — Responsive ចាំបាច់តែ Web Application ។

CSS — Media Queries for Responsive Layout 
/* Desktop First — Default Styles */
.app-layout {
  display: grid;
  grid-template-columns: 220px 1fr;  /* Sidebar + Content */
}

/* Tablet — 768px to 1023px */
@media (max-width: 1023px) {
  .app-layout {
    grid-template-columns: 1fr;       /* Sidebar hidden */
  }
  .sidebar { display: none; }
  .hamburger-btn { display: flex; }     /* Show menu toggle */
}

/* Mobile — below 768px */
@media (max-width: 767px) {
  .card-grid-3 {
    grid-template-columns: 1fr;       /* Stack cards */
  }
  .pos-layout {
    flex-direction: column;          /* Stack POS panels */
  }
}
✅
Mobile-First vs Desktop-First
Mobile-First: Design Mobile ជាមុន → បន្ថែម Layout ធំ ។ Google, Bootstrap ប្រើ Approach នេះ ។ Desktop-First: Design Desktop ជាមុន → Shrink ។ SBCMS Desktop App ប្រើ Desktop-First ។ Web App ត្រូវ Mobile-First ព្រោះ Customer Order ពី Phone ។

Section 06

Form Design Best Practices

Form ជា Element ដែល User ប្រៀបធៀបច្រើនជាង ក្នុង App ។ Login Form, Order Form, Employee Form — Form ល្អ = User Fill-out លឿន, Error តិច ។

✅ DO — Form ល្អ
  • Label នៅ លើ Input (Top-aligned)
  • ប្រើ Placeholder ជា Hint ប្រហែល
  • Group Fields ដែលទាក់ទងគ្នា
  • Validation ភ្លាមៗ (Inline Error)
  • ប្រើ Auto-focus Field ដំបូង
  • Button "Submit" ច្បាស់, CTA Strong
  • Required Field មាន * ទំនោររៀបរាប់
❌ DON'T — Form ខ្សោយ
  • Label ខាងឆ្វេង, Input ខាងស្ដាំ (ងាយ Read)
  • Placeholder ជា Label (disappears on type)
  • Fields ច្រើន Column ក្នុង Mobile
  • Error បង្ហាញតែ Submit (late feedback)
  • ប្រើ Reset Button — User ចុចខុស Clear ទាំងអស់
  • Button "Submit" — Generic, ខ្វះ Context
  • Fields ច្រើនពេក — Cognitive Overload

Section 08

Accessibility (A11y) — ការ Design ដែលរួមបញ្ចូលអ្នកទាំងអស់

Accessibility ឬ A11y ជា Practice ធ្វើ App ដែល User ទាំងអស់ប្រើបាន រួមមាន User ដែលមានបញ្ហាភ្នែក, សសៃ, ការសោត ។ អ្នកច្បាប់ UN-CRPD តម្រូវ Digital Product ត្រូវ Accessible ។

❌ Inaccessible
  • Button ដែលមាន Icon ប៉ុណ្ណោះ — Screen Reader អានអ្វី?
  • Color ប្រើដើម្បី Signal Error — Color Blind User មើលមិនឃើញ
  • Font តូចជាង 12px — ម្នាក់ចង្ងាញ់អានពិបាក
  • Click Target ≤ 24px — ម្នាក់ទៅ Motor Impairment ចុចខុស
  • Image គ្មាន Alt Text
✅ Accessible
  • Button មាន aria-label — Screen Reader អានបាន
  • Error = Color + Icon + Text — Color Blind ក៏ដឹង
  • Font ≥ 14px, Body Text ≥ 16px
  • Click/Touch Target ≥ 44×44px (Apple HIG)
  • Image មាន alt="..."

SBCMS Applied — ការ Apply ជាក់ស្ដែង

តោះ Summarize ការ Apply UI/UX Principles ទៅ SBCMS Wireframes (Task #23–27) ។

Task #24 — POS Order Form

UI/UX Analysis: POS Order

  • Grid Layout: Flexbox Split — Menu Grid (flex:1) + Order Panel (280px fixed) → 2-Zone Layout
  • Component States: Menu Item Card — Default / Hover (Caramel border) / Selected (Active bg)
  • User Flow: Select Item → Add to Panel → Adjust Qty → Confirm → DB Save → Stock Deduct
  • Feedback: Real-time Total update → Visibility of System Status (Nielsen Rule #1)
  • Accessibility: +/− Qty Buttons ≥ 44px Touch Target, Clear Labels
Task #25 — Inventory Form

UI/UX Analysis: Inventory

  • Alert System: Warning Banner ជា Visual Cue — User ដឹង Low Stock ភ្លាម
  • Color Coding: 🔴 Danger = Low Stock, 🟡 Warning = Medium, 🟢 Success = Healthy
  • Progress Bar: Visual Stock Level — ល្អជា Number ចំពោះ Quick Scan
  • IA: Filter (Status) + Search → User ស្វែងរក Item ក្នុង 2 Steps
  • Data Table: Header Sort-able, Row Hover State, Action Buttons aligned right
Task #26 — Report Form

UI/UX Analysis: Reports

  • Period Selector: Tab Style (Daily/Weekly/Monthly) → IA Clear, Quick Switch
  • 4 KPI Cards: Information Hierarchy — Revenue (Most Important) → Orders → Avg → Top Item
  • Bar Chart: Data Visualization ជា Visual → User Grasp Trend ក្នុង Seconds
  • Export Button: Affordance — ប្រើ Icon ⬇ + Label "Export PDF" → ច្បាស់
  • Responsive: Chart + Table side-by-side Desktop, Stack on Mobile
