Section 01

UI និង UX គឺជាអ្វី?

នៅក្នុងពិភព Software Development, យើងតែងតែឮពាក្យ UI និង UX រួមគ្នា ប៉ុន្តែទាំងពីរនេះមានន័យខុសគ្នា។ ការយល់ដឹងអំពីភាពខុសគ្នានេះ គឺជាជំហានដំបូងក្នុងការក្លាយជា Designer ឬ Developer ល្អ។

🖼️
UI — User Interface
រូបរាង · ពណ៌ · ប្លង់
UI ជាអ្វីដែល មើលឃើញ — Buttons, Colors, Fonts, Icons, Layout
ចុចប៊ូតុងមួយ → ពណ៌ប្រែ, Animation លេចឡើង — នោះជា UI
ផ្តោតលើ Visual Design — ស្អាត, មានរចនា, ទាក់ទាញ
ឧបករណ៍: Figma, Adobe XD, CSS, HTML
🧠
UX — User Experience
ប្រសិទ្ធភាព · ភាពងាយ · អារម្មណ៍
UX ជា អារម្មណ៍ ដែល User មានពេលប្រើ App — ងាយ? ច្របូកច្របល់? រំភើប?
User ចូល App ហើយស្វែងរកបានលឿន, មិនច្របូកច្របល់ — នោះជា UX ល្អ
ផ្តោតលើ User Flow, Usability, Research
ឧបករណ៍: Personas, Wireframes, User Testing
☕
ពាក្យប្រៀបធៀបដ៏ល្អ — Coffee Shop
ស្រមៃថា Sharp Bean Café — UI ដូចជាការតុបតែង cafe: ជញ្ជាំងស្អាត, menu board ច្បាស់, ពណ៌ Espresso ស្រស់ស្អាត ។ UX ដូចជាដំណើរការ: Order លឿន, Barista យល់ Order ត្រូវ, អាហារមកឆាប់, Customer ពេញចិត្ត — ហើយត្រឡប់មកវិញ ។
Section 02

ភាពខុសគ្នា: UI vs UX

UI និង UX ធ្វើការជាមួយគ្នា ប៉ុន្តែមានទំនួលខុសត្រូវខុសគ្នា។ Product ល្អត្រូវការ ទាំង UI ស្អាត ទាំង UX ល្អ ។

លក្ខណៈ UI UX
ផ្តោតលើ Visual, Aesthetics Usability, Behavior
សំណួរ "App ស្អាតទេ?" "User ប្រើបានងាយទេ?"
លទ្ធផល Mockups, Style Guide Wireframes, User Flow
ដំណើរការ Color, Font, Layout Research, Testing, Iteration
SBCMS Design System, WinForms Style Order Flow, Login Flow
⚠️
កំហុស Common
UI ស្អាត ≠ UX ល្អ ! App អាចមានពណ៌ស្អាត, Font ល្អ ប៉ុន្តែ User ស្វែងរកអ្វីមិនឃើញ — នោះ UX ខ្សោយ ។ ផ្ទុយទៅវិញ App ងាយប្រើ ប៉ុន្តែបើរូបរាងអាក្រក់ (ugly) — UI ខ្សោយ ។ ត្រូវការ ទាំងពីរ
Section 03

ហេតុអ្វីត្រូវរៀន UI/UX?

ក្នុងការសិក្សា Software Engineering, UI/UX ជា Phase ដ៏សំខាន់ ព្រោះ System ល្អត្រូវ ទាំង Code ល្អ ទាំង Interface ល្អ ។ User ស្ទើរមិនដែលអាន Source Code — ពួកគេឃើញ UI ប៉ុណ្ណោះ ។

💼
Professional Skills
ក្រុមហ៊ុន IT ទាំងអស់ ត្រូវការ Developer ដែលយល់ Design ។
👤
User-Centered
រៀន Design ជួយ Developer គិតដូច User — ធ្វើ Feature ត្រឹមត្រូវ ។
🚀
Product Quality
UI/UX ល្អ → User ពេញចិត្ត → Product ជោគជ័យ ។
💡
នៅក្នុង SBCMS Project
Team យើងបានបង្កើត Design System (Task #21, #22), Wireframes (Task #23–27) ដែលផ្អែកលើ UI/UX Principles ។ នេះ មិនមែន "ពណ៌ស្អាត" ប្រចាំ — វាជា System ដែល Scalable, Consistent ។
Section 04

គោលការណ៍ UI/UX ជាមូលដ្ឋាន

Jakob Nielsen ដែលជា UX Expert ល្បីល្បាញ បានបង្កើត 10 Usability Heuristics ។ ខាងក្រោមនេះ ជា 6 គោលការណ៍ដ៏ចាំបាច់ ។

👁️
1. Visibility of System Status
ប្រាប់ User ថា System ធ្វើអ្វី ។ ឧ. Loading Spinner ពេល Submit Order, Badge "Confirmed" ពេល Order ជោគជ័យ ។
🌍
2. Match Real World
ប្រើ Language និង Icon ដែល User ស្គាល់ ។ ប្រើ 🗑️ សម្រាប់ Delete, ✏️ សម្រាប់ Edit — មិនប្រើ Technical Code Words ។
🎮
3. User Control & Freedom
User ជ្រើស Cancel, Back, Undo បាន ។ ឧ. ប៊ូតុង "Clear" ក្នុង POS Form ដូចក្នុង SBCMS Wireframe Task #24 ។
📐
4. Consistency
ប្រើ Button Style, Font, Color ដូចគ្នា ។ SBCMS Design System ជា Consistency Guide — ប្រើ Espresso Brown ជារបស់ Brand ។
🛡️
5. Error Prevention
ការពារ Error ជាការប្រសើរជាង Error Message ។ ឧ. Disable ប៊ូតុង "Confirm" ពេល Cart ទទេ ។
🧹
6. Aesthetic & Minimalism
បង្ហាញតែ Information ចាំបាច់ ។ Information ច្រើនពេក → User Overwhelmed → UX ខ្សោយ ។

Color Theory — ទ្រឹស្ដីពណ៌

ពណ៌ជា Element សំខាន់បំផុតក្នុង UI ។ ពណ៌ប៉ះពាល់ដល់ អារម្មណ៍, ការទុកចិត្ត, ការសម្រេចចិត្ត របស់ User ។

SBCMS Design System — Color ត្រូវ
SBCMS Design System ប្រើ Espresso ជា Header/Nav, Cream ជា Background, Caramel ជា Accent CTA — ដូច 60-30-10 Rule ។ ពណ៌ទាំងអស់ Consistent តាមរយៈ CSS Variables ។

Color Contrast — WCAG

WCAG (Web Content Accessibility Guidelines) កំណត់ថា Text ត្រូវ Contrast Ratio ≥ 4.5:1 ។ នេះ ទាក់ទងនឹង Accessibility — User ដែលមានបញ្ហាភ្នែកក៏អានបាន ។

CSS Variables — SBCMS Color System 
:root {
  /* Primary Palette */
  --espresso:  #2C1A0E;   /* Darkest — Headers, Primary Text */
  --roast:      #4A2C0A;   /* Brand Primary — Nav Background */
  --caramel:    #C89B3C;   /* Gold Accent — CTA Buttons ✦ */
  --cream:      #FDF6EC;   /* Page Background */

  /* Semantic Colors */
  --success:    #3A7D44;   /* Done, Confirmed */
  --warning:    #C47C1A;   /* Low Stock, Pending */
  --danger:     #B83232;   /* Error, Delete */
}
Section 06

Typography — ការជ្រើស Font

Typography គឺជា ជំនាញប្រើ Font ក្នុង Design ។ Font ជ្រើសខុស → UI មើលទៅ Amateur ។ Font ជ្រើសត្រូវ → Professional, ទំនុកចិត្ត ។

Font Pairing — ការដាក់ Font ជាគូ

Designer ល្អ ច្រើន Pair Font ១ Display + ១ Body ។ Display Font = ស្រស់, Character; Body Font = ងាយអាន ។

Typography Rules

1
Font Size Hierarchy
H1 > H2 > H3 > Body > Label — ធ្វើឱ្យ User ដឹង Information ណា Important ។
2
Line Height (Leading)
Body Text ត្រូវ Line Height ប្រហែល 1.6–1.8 — ងាយអាន, ចន្លោះល្អ ។
3
Maximum 2–3 Fonts
Font ច្រើនពេក → ច្របូកច្របល់ ។ SBCMS ប្រើ Playfair Display + DM Sans + DM Mono — ត្រឹមបី ។
4
Monospace for Code/Data
Consolas, DM Mono — ប្រើសម្រាប់ Order ID, Code, Database Values — ងាយអានជា Column ។

Section 08

Tools & UX Design Workflow

Designer ប្រើ Tools ខុសគ្នា ។ SBCMS Team ប្រើ Figma ដើម្បីធ្វើ Wireframe, draw.io ដើម្បីធ្វើ Diagram, HTML/CSS ដើម្បីធ្វើ Design System ។

Figma draw.io Visual Paradigm Adobe XD Sketch HTML/CSS React Tailwind CSS UI Designer UX Researcher Frontend Dev

UX Design Process

1
Research — ស្រាវជ្រាវ
ស្គាល់ User: ពួកគេជានរណា? ចង់ធ្វើអ្វី? Problem អ្វី? → Personas, User Stories, Interviews ។
2
Wireframe — គ្រោង
គ្រោង Layout ប្រហែលៗ — ដោយ Pencil ឬ Tool ។ SBCMS: Task #23–27 ជា Wireframes ។
3
Prototype — គំរូ
ធ្វើ Clickable Prototype ដែល User អាច Test — ស្វែងរក Problem មុន Code ។
4
Test & Iterate — ធ្វើ Test
User Test → ស្ដាប់ Feedback → កែ Design → Test ម្ដងទៀត — Cycle ។
5
Handoff to Developer
ប្រគល់ Design + Design System ដល់ Developer ដើម្បី Code ។ SBCMS: Design System + Wireframes → C# WinForms ។
Section 09

ឧទាហរណ៍: SBCMS UI/UX Analysis

តោះ Analyze UI/UX ក្នុង SBCMS Wireframes ដែល Team បានបង្កើត ។

SBCMS — Task #23: Login Form

UI Analysis: Login Form

  • Color: Dark Espresso Background → Professional, Secure, Brand Consistent
  • Layout: Centered Card → ផ្តោត User Attention, ងាយប្រើ
  • Role Selector: Employee / Manager Toggle → Clear Role-Based Access
  • Typography: Playfair Display + DM Mono → Elegant + Technical
  • UX Flow: Login → Redirect to Dashboard ដោយ Role → Efficient
SBCMS — Task #24: POS Order Form

UX Analysis: POS Order Form

  • Split Layout: Menu Grid + Order Panel → ឃើញ Menu និង Order ក្នុងតែ Screen ១
  • Category Tabs: ចុច Filter → ស្វែងរក Item លឿន (UX Principle #3: Control)
  • Real-time Total: Total Update ដោយ Auto → (**Visibility of System Status**)
  • +/− Qty Buttons: User ចង្អុលបន្ថែម/កាត់ ងាយ → (User Control & Freedom)
  • Confirm Button: Accent Color Caramel → ចូលភ្នែក, CTA ច្បាស់
🔗
ភ្ជាប់ UI/UX → Backend
UI/UX ល្អ → User Order ផ្ញើ Request → Backend (OrderService.ConfirmOrder()) → Database (sp_CreateOrder) → Stock Deducted (InventoryService.DeductStock()) → Alert ត្រឡប់ UI → ✅ នេះ ជា End-to-End Flow ដែលចាប់ផ្ដើម UX ។
